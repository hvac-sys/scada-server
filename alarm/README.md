# SCADA Alarm Engine Module (`alarm`)

이 모듈은 수집된 센서 데이터를 실시간으로 감시하고, 사전 정의된 룰(Rule)에 따라 알람 발생 및 복구를 판정하는 핵심 논리 제어 모듈입니다.

## 1. 주요 책임 (Responsibilities)

- `acquisition` 모듈에서 발행한 데이터 이벤트를 리스닝하여 설정된 임계값과 비교.
- 일시적인 노이즈로 인한 오작동 방지를 위한 **Delay(지연 시간)** 및 **Hysteresis(데드밴드)** 적용.
- 알람 상태 전이(State Machine) 관리: `NORMAL` -> `OCCURRED` -> `ACKNOWLEDGED` -> `CLEARED`.
- 상태 변경 시 알람 이벤트를 발행하여 `historian`에 저장 요청 및 `websocket`에 푸시 요청.

## 2. 모듈 의존성 (Dependencies)

* `core` (이벤트 및 공통 DTO)

## 3. 알람 로직 및 정책 (Alarm Policies)

### 3.1. 알람 임계값 처리 로직 (Hysteresis & Delay)

단순히 임계값을 넘었다고 바로 알람을 띄우지 않습니다. 산업 현장의 노이즈 방지를 위해 아래 두 가지를 적용합니다.

* **Delay (지연 시간)**: 
  * 설정값 초과 상태가 `N초` 이상 지속되어야만 알람 발생 (`OCCURRED`).
  * *예: 온도 > 30℃ 상태가 5초 유지되면 알람.*
* **Hysteresis (데드밴드)**: 
  * 알람 복구(`CLEARED`) 시, 발생 임계값보다 지정된 범위만큼 여유를 두고 떨어져야 복구로 판정. (헌팅 방지)
  * *예: 상한 알람 30℃ 초과, Hysteresis 2℃ -> 28℃ 밑으로 내려와야 알람 복구.*

### 3.2. 알람 등급 (Severity Levels)

| 등급 | 코드 | 내용 | 처리 권장 사항 |
|---|---|---|---|
| **정보** | `INFO` | 단순 시스템 이벤트 (로그인, 수동 모드 전환) | 로그만 저장, 화면 팝업 없음 |
| **경고** | `WARNING` | 설정된 목표 범위를 벗어남 (예: 온도 ±2℃ 편차 발생) | 알람 리스트 표시 (황색), 동작 유지 |
| **위험** | `CRITICAL`| 장비 보호 한계치 도달 (예: 온도 +5℃ 초과) | 알람 팝업 띄움 (적색), 시스템 안전 모드 강제 전환 |
| **장애** | `FAULT` | 통신 끊김, 센서 단선, E-Stop 눌림 | 사이렌 울림, 즉시 전체 장비 정지 |

### 3.3. 알람 상태 관리 (State Transitions)

1. **NORMAL** (정상): 장비 정상 동작 중.
2. **OCCURRED** (발생): 조건 만족 시 발생. HMI에 빨간불 점멸 시작.
3. **ACKNOWLEDGED** (확인): 운전원이 HMI에서 '알람 확인' 버튼 클릭. 점멸 멈추고 켜진 상태 유지 (누가 확인했는지 ID 기록).
4. **CLEARED** (복구): 장비 상태가 정상 수치로 돌아옴. 이력 목록으로 넘어감.

## 4. 데이터베이스 연동 주의사항

이 모듈 자체는 DB에 직접 접근하지 않습니다 (도메인 순수성 유지). 
알람 룰(Alarm Rule)은 메모리에 캐싱하여 1초 주기 검사를 빠르게 수행하며, 룰의 CRUD 작업이나 알람 이력(History) 저장은 `historian` 또는 `api` 모듈에서 처리합니다.
